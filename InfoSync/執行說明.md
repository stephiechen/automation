# InfoSync 執行說明

## 系統整體架構

```
使用者（瀏覽器）
      │
      │  填寫表單 → 按下「同步」
      ▼
  app.py（Flask 後端，本機 port 5000）
      │
      ├──► api/shopline.py ──► Shopline API
      │         更新官網頁面 HTML
      │
      ├──► api/google_business.py ──► Google Business Profile API
      │         更新營業時間 / 發貼文
      │
      └──► api/meta.py ──► Meta Graph API
                發文到 Facebook + Instagram
```

---

## 啟動流程

```
1. 填好 .env（憑證）
2. python app.py
3. 開啟瀏覽器 → http://localhost:5000
4. 選 Tab → 填資料 → 勾選平台 → 送出
5. 畫面顯示各平台成功 ✓ / 失敗 ✗
```

## 三個功能的執行流程

---

### 功能 A｜醫師資訊同步

**適用情境**：新進醫師上線、更新醫師照片或學經歷

**表單欄位**
| 欄位 | 說明 |
|------|------|
| 姓名 | 顯示於醫師卡片 |
| 科別 | 如：家醫科、小兒科 |
| 學經歷 / 簡介 | 自由輸入文字 |
| 醫師照片 | 上傳圖片檔（jpg / png） |
| 同步平台 | 勾選：官網（醫師陣容頁） |

**執行步驟**

```
Step 1  使用者填寫表單 → 按「同步」
Step 2  瀏覽器 POST /sync/doctor（multipart/form-data）
Step 3  app.py 接收資料：
          - 圖片存到 InfoSync/uploads/ 資料夾
          - 讀取 SHOPLINE_DOCTOR_PAGE_ID（from .env）
Step 4  api/shopline.py：
          GET  /pages/{DOCTOR_PAGE_ID}.json
              → 取得醫師陣容頁現有 HTML
          組合新醫師卡片 HTML（姓名 + 科別 + 簡介 + 圖片）
          PUT  /pages/{DOCTOR_PAGE_ID}.json
              → 將新卡片附加到現有 HTML 後方
Step 5  回傳結果 → 瀏覽器顯示：
          官網 ✓  或  官網 ✗（附錯誤訊息）
```

**Shopline 更新的實際內容**

醫師陣容頁的 `body_html` 欄位會新增：
```html
<div class="doctor-card">
  <img src="/uploads/photo.jpg" alt="王小明" />
  <h3>王小明</h3>
  <p class="specialty">家醫科</p>
  <p class="bio">台大醫學系畢業...</p>
</div>
```

### 功能 B｜營業時間同步

**適用情境**：診所調整門診時間、假日停診

**表單欄位**
| 欄位 | 說明 |
|------|------|
| 週一～週日 開診時間 | time picker |
| 週一～週日 關診時間 | time picker |
| 休診勾選 | 打勾 = 不開診 |
| 同步平台 | 勾選：官網（診所資訊頁）/ Google 商家 |

**執行步驟**

```
Step 1  使用者設定每天時間 → 按「同步」
Step 2  瀏覽器 POST /sync/hours
Step 3  app.py 整理成 hours dict：
          {
            "monday":    {"open": "09:00", "close": "17:00", "closed": false},
            "saturday":  {"open": "09:00", "close": "12:00", "closed": false},
            "sunday":    {"closed": true},
            ...
          }

【Shopline 分支】
Step 4a api/shopline.py：
          將 hours dict 轉成 HTML 表格
          PUT /pages/{INFO_PAGE_ID}.json
              → 更新診所資訊頁的 body_html

【Google 分支】
Step 4b api/google_business.py：
          將 hours dict 轉成 GMB regularHours 格式：
          [
            { "openDay": "MONDAY", "openTime": {"hours":9},
              "closeDay": "MONDAY", "closeTime": {"hours":17} },
            ...
          ]
          PATCH https://mybusinessbusinessinformation.googleapis.com/v1/{location}
              updateMask: "regularHours"
              → Google 商家後台立即反映新時間

Step 5  回傳結果 → 瀏覽器顯示各平台結果
```

### 功能 C｜公告 / 貼文同步

**適用情境**：健康衛教文、活動通知、節慶公告

**表單欄位**
| 欄位 | 說明 |
|------|------|
| 內容 | 貼文文字（必填） |
| 圖片 | 上傳圖片（Instagram 必填） |
| 同步平台 | 勾選：Google 商家 / Facebook / Instagram |

**執行步驟**

```
Step 1  使用者填寫內容 + 上傳圖片 → 按「同步」
Step 2  瀏覽器 POST /sync/post
Step 3  app.py：圖片存到 uploads/

【Google 分支】
Step 4a api/google_business.py：
          POST .../localPosts
          body: { "summary": "...", "media": [{"sourceUrl": "..."}] }
          → Google 商家「最新動態」出現貼文

【Facebook 分支】
Step 4b api/meta.py：
          有圖片 → POST /{page_id}/photos  (caption + url)
          無圖片 → POST /{page_id}/feed    (message)
          → Facebook 粉絲頁發文

【Instagram 分支】
Step 4c api/meta.py（兩步驟）：
          Step ① POST /{ig_user_id}/media
                 body: { image_url, caption }
                 → 取得 creation_id
          Step ② POST /{ig_user_id}/media_publish
                 body: { creation_id }
                 → IG 發文完成
          ※ 無圖片時直接回傳錯誤，不呼叫 API

Step 5  回傳各平台結果：
          Google ✓  Facebook ✓  Instagram ✗（若無圖）
```

## 認證機制說明

| 平台 | 認證方式 | Token 存放位置 |
|------|---------|--------------|
| Shopline | `X-Shopline-Access-Token` Header | `.env` |
| Google | Service Account JSON（OAuth 2.0） | `service_account.json` 檔案路徑存 `.env` |
| Meta (FB/IG) | Page Access Token（長效 60 天） | `.env` |

### Token 過期處理
- **Shopline**：Token 長效，不需更新
- **Google**：Service Account 自動 refresh，不會過期
- **Meta**：60 天過期，需手動到 Meta Developer 重新換發長效 Token

## 錯誤處理邏輯

每個平台的 API 呼叫都是**獨立 try/except**，一個平台失敗不影響其他平台：

```python
# 例：Google 失敗，不影響 Facebook 繼續執行
if "google" in platforms:
    try:
        create_post(message, image_url)
        results["google"] = {"ok": True}
    except Exception as e:
        results["google"] = {"ok": False, "error": str(e)}  # ← 記錄錯誤

if "facebook" in platforms:
    try:
        post_to_facebook(message, image_url)                 # ← 繼續執行
        results["facebook"] = {"ok": True}
    except Exception as e:
        results["facebook"] = {"ok": False, "error": str(e)}
```

前端顯示效果：
```
Google ✗ 401 Token 過期   Facebook ✓   Instagram ✓
```

## 上線前必做事項

### 1. 取得 Shopline Page ID
```bash
curl -H "X-Shopline-Access-Token: 你的TOKEN" \
  https://你的STORE.myshopline.com/admin/openapi/v20230601/pages.json
```
找到「醫師陣容」和「診所資訊」兩頁的 `id` → 填入 `.env`

### 2. 取得 Google Location Name
格式：`accounts/123456789/locations/987654321`
→ GCP Console 或 Google Business Profile API Explorer 可查詢

### 3. 取得 Meta 長效 Page Access Token
1. [Meta for Developers](https://developers.facebook.com/tools/explorer/) → Graph API Explorer
2. 選粉絲頁 → 取得短效 Token
3. 換成長效（60 天）Token：
   ```
   GET /oauth/access_token
     ?grant_type=fb_exchange_token
     &client_id={APP_ID}
     &client_secret={APP_SECRET}
     &fb_exchange_token={短效TOKEN}
   ```

### 4. 取得 Instagram User ID
```
GET /me/accounts?access_token={PAGE_TOKEN}
```
從 `connected_instagram_account.id` 取得 IG User ID
